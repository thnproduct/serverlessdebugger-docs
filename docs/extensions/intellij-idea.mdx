# IntelliJ IDEA
frogate's IntelliJ IDEA plugin allows you to debug your AWS Lambda functions written in [Java](/configuration/java). 
With a small configuration, debugging is as simple as a click of a button.

## Introduction 
Start online debug sessions while the AWS Lambda functions run on AWS Cloud. Thundra sets up an easy and secure 
connection between your AWS Lambda functions and IntelliJ IDEA. Debug your serverless applications natively with their own permissions. Stop mocking resources and simulating 
the environments! Put breakpoints and pause the execution of the [Java](/configuration/java) functions to see the stack trace.

## Installation
In order to use frogate's IntelliJ IDEA plugin, you need to **complete** all installation steps. Please follow the instructions below.

<Container>

    <li> 
      Click on the "+" sign and select <strong>Thundra Debugger</strong> from the list. Paste the auth token to the <strong>Authentication Token</strong> field. Enter a name for this new profile and then save.
    </li>
  </ul>
  </Step>
  <Step step="4" stepHeader="Edit AWS variables">
    Go to your <strong>AWS Console</strong> and set your auth token value with the <code>THUNDRA_AGENT_LAMBDA_DEBUGGER_AUTH_TOKEN</code> key as your environment variable.
    <br/><br/>
    <img src='/img/awsconsole-authtoken.png' alt="AWS Console" />
  </Step>  
  <Step step="5" stepHeader="Instrument your function">
    After installing the plugin, instrument your AWS Lambda. You can instrument your <a href="/configuration/java">Java</a> functions to use IntelliJ IDEA plugin.
  </Step>
</Container>
