Serverless Debugger's VS Code extension allows you to debug your AWS Lambda functions written in [NodeJS](/configuration/nodejs) and [Python](/configuration/python). With a small configuration, debugging is as simple as a click of a button.

## Introduction 
Start online debug sessions while the AWS Lambda functions run on AWS Cloud. Thundra sets up an 
easy and secure connection between your AWS Lambda functions and VSCode. Debug your serverless 
applications natively with their own permissions. Stop mocking resources and simulating the environments! 
Put breakpoints and pause the execution of the [NodeJS](/configuration/nodejs) and [Python](/configuration/python) functions to see the stack trace.

## Commands

- ```ServerlessDebugger: Start Debugger``` to start the debugger with the current profile's settings
- ```ServerlessDebugger: Start Debugger (with a session name)``` to start the debugger with the given session name
- ```ServerlessDebugger: Change active profile``` to change the active profile to the one given in your config file
- ```ServerlessDebugger: Edit configuration``` to edit the JSON file containing your debugger configuration

You can access all of the above commands from the command palette ```(Cmd+Shift+P or Ctrl+Shift+P)```.

## Self-Hosted Broker

A self-hosted instance of the broker that provides the necessary functionality for online debugging can be installed 
on your own AWS account using our CloudFormation template or our CDK application. Please follow [this guide](/self-hosted-broker/CloudFormation) to learn more.
